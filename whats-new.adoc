== 06 abril 2026



=== Amazon FSX para ONTAP de NetApp



==== Actualizaciones de análisis bien diseñadas

Workload Factory analiza tus sistemas de archivos FSx para ONTAP en busca de *volúmenes NAS inactivos*. El análisis de configuración identifica los volúmenes NAS que no se utilizan activamente y recomienda acciones para optimizar la utilización del almacenamiento y reducir costos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Implementa sistemas de archivos FSx para ONTAP bien diseñados"]



==== Función de tabla de registro diario para puntos de acceso S3

La función de tabla de diario proporciona una forma de gestionar el almacenamiento de objetos S3 de FSx ONTAP a escala, permitiéndote auditar fácilmente los eventos de acceso de usuario y las operaciones en los objetos de tus buckets. Con Workload Factory, puedes configurar exploraciones automáticas de tus buckets y tablas S3 para mantener un registro de los eventos, mostrando todos los objetos, sus detalles y las acciones de los usuarios.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/setup-journal-table.html["Configura la infraestructura de la tabla de registro en diario para NetApp Workload Factory"]



==== Soporte para la gestión de los igroups existentes

Workload Factory admite la gestión de igroups existentes. Puedes añadir o eliminar dispositivos de bloque e iniciadores de host para entregar a tiempo tu carga de trabajo desplegada.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Administra el almacenamiento en bloques en NetApp Workload Factory"]



==== Compatibilidad para especificar el agregado para volúmenes FlexVol

Workload Factory permite crear volúmenes FlexVol en un agregado elegido. Esto te permite dividir la carga de capacidad y rendimiento entre todos los recursos del sistema de archivos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-volume.html["Crea un volumen FlexVol en NetApp Workload Factory"]



==== Último tiempo de acceso para dispositivos de bloque

Workload Factory informa del último tiempo de acceso a los dispositivos de bloque. El tiempo se basa en la última vez que accediste al igroup en Workload Factory. Saber cuándo se utilizó por última vez cada dispositivo de bloque te permite optimizar tus recursos.



=== Cargas de trabajo de base de datos



==== Análisis bien diseñado para Oracle

El análisis bien diseñado incluye las siguientes nuevas configuraciones para Oracle:

* *Oracle Critical Patch Updates*: evalúa los parches de seguridad que faltan para las instalaciones de bases de datos autogestionadas de Oracle compatibles.
* *Cross-region replication*: comprueba si la replicación entre regiones (CRR) está activada para tu FSx for ONTAP file systems que dan servicio a bases de datos Oracle.
* *Instantáneas consistentes con las aplicaciones*: comprueba si la base de datos Oracle está protegida por instantáneas consistentes con las aplicaciones con NetApp SnapCenter.


link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Implementar configuraciones de base de datos bien diseñadas en Workload Factory"]



==== Evaluación única para Oracle

Workload Factory ahora ofrece una evaluación única para analizar las configuraciones de tus despliegues de bases de datos Oracle cuando lo necesites: nuevos despliegues, posmigración, solución de problemas y más. Recibirás recomendaciones para ayudarte a optimizar las configuraciones de tus despliegues y cumplir con las mejores prácticas. Esta es tu oportunidad de usar la función well-architected sin comprometerte completamente con el proceso de configuración. Puedes evaluar tus bases de datos sin almacenar credenciales en Workload Factory, registrar recursos de bases de datos ni configurar conexiones externas.

link:https://docs.netapp.com/us-en/workload-databases/optimize-configurations.html["Realiza una evaluación única de tus implementaciones de bases de datos Oracle"]



==== Mayor ahorro de costes en varias bases de datos Oracle

Hemos mejorado el análisis de ahorro de costes para Oracle con el fin de ofrecer una solución de almacenamiento consolidada para varias bases de datos locales de Oracle en un único sistema de archivos FSx para ONTAP. Esta consolidación optimiza los costes de almacenamiento al reducir el número de sistemas de archivos necesarios para varias bases de datos, lo que se traduce en un mayor ahorro de costes.

link:https://docs.netapp.com/us-en/workload-databases/explore-savings.html#explore-savings-for-detected-hosts["Explora el ahorro para las bases de datos detectadas en Oracle on-premises"]

=== Cargas de trabajo EDA



==== Recomendaciones de capacidad de EDA

Workload Factory ahora analiza automáticamente la capacidad SSD de cada clúster FSx para ONTAP en tu inventario EDA y genera recomendaciones de capacidad cuando se necesitan ajustes. El algoritmo se ejecuta una vez cada 24 horas para cada clúster.

Cuando se identifica una recomendación:

* Recibes una notificación inmediata por correo electrónico o WAD (no forma parte del resumen semanal).
* Aparece un indicador luminoso en el cluster afectado en la tabla Clusters.
* Un resumen en el panel de descripción general muestra el número total de clusters con recomendaciones activas.


Selecciona un cluster con una recomendación activa para ver la recomendación, que explica el cambio sugerido y el razonamiento.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Más información sobre las recomendaciones de capacidad"^].



==== Análisis automatizado de latencia de IA

El análisis de latencia ahora incluye el análisis de agentes de IA.

Para escenarios complejos que impliquen latencia de datos o de clúster, puedes invocar el análisis de agentes de IA para recibir:

* Identificación detallada de la causa raíz
* Lista de clientes EC2 afectados
* Medidas correctoras específicas


El análisis del agente de IA requiere la configuración de ARN del modelo de Amazon Bedrock y está disponible bajo demanda para escenarios en los que el análisis básico recomienda una investigación más profunda. El agente de IA puede identificar problemas como volúmenes bully, configuraciones de puntos de montaje no óptimas, necesidades de reequilibrio de FlexGroup y requisitos de escalado horizontal.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Más información sobre el análisis de agentes de IA"^].



== 09 de marzo de 2026



=== Cargas de trabajo de



==== Presentamos la compatibilidad del asesor de migración para Google Cloud VMware Engine

NetApp Workload Factory ahora es compatible con Google Cloud VMware Engine. Puedes migrar tus cargas de trabajo VMware locales a Google Cloud VMware Engine usando Google Cloud NetApp Volumes como almacenes de datos NFS externos, optimizando los costes y permitiendo más control sobre tu entorno VMware sin necesidad de refactorizar o cambiar de plataforma tus aplicaciones.

El asesor de migración de Google Cloud VMware Engine proporciona:

* Dimensionamiento y diseño automatizados del clúster Google Cloud VMware Engine con compatibilidad con los tipos de nodo VE1 y VE2
* Optimización de la disposición de las máquinas virtuales entre los datastores internos de vSAN y los datastores NFS externos de Google Cloud NetApp Volumes.
* Comparación detallada de costes y análisis del coste total de propiedad entre las configuraciones solo vSAN y Google Cloud NetApp Volumes-enhanced.
* Soporte para los niveles de servicio Standard, Premium y Extreme de Google Cloud NetApp Volumes.
* Despliegue de Infrastructure-as-Code con Terraform para el aprovisionamiento automatizado.

=== Amazon FSX para ONTAP de NetApp



==== Actualizaciones de análisis bien diseñadas

Workload Factory analiza sus sistemas de archivos FSx for ONTAP para las siguientes configuraciones:

* Capacidad SSD infrautilizada: comprueba si el nivel de capacidad SSD está infrautilizado y recomienda reducir la capacidad SSD para optimizar costos.
* Programa backups de volúmenes: Workload Factory mejora la recomendación de configuración y añade opciones para activar FSx for ONTAP backup o AWS Backup. FSx for ONTAP backup aplica una política predeterminada a todos los volúmenes en un sistema de archivos, creando backups diarios que se conservan hasta 90 días. AWS Backup te permite elegir diferentes periodos de retención para cada volumen en el plan de backup.




==== Workload Factory admite el caso de uso de IA y ML para la replicación

Workload Factory admite replicar datos ONTAP on-premises en FSx para ONTAP y crear puntos de acceso S3 para los servicios de AWS Artificial Intelligence (AI) y machine learning (ML) para acelerar los flujos de trabajo de AI/ML.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replica datos locales para casos de uso de AI y ML"]



==== Editar puntos de acceso S3 existentes

Puedes cambiar el usuario y el tipo de usuario para un punto de acceso S3 existente adjunto a un volumen. También puedes activar o desactivar los metadatos para el punto de acceso.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/edit-s3-access-points.html["Editar puntos de acceso S3 en NetApp Workload Factory"]



==== Diagrama de almacenamiento por bloques añadido

Si estás usando almacenamiento en bloque, puedes ver el diagrama de almacenamiento en bloque para visualizar cómo los dispositivos de bloque se conectan a los nodos e igroups. El diagrama te ayuda a entender y gestionar tu almacenamiento en bloque.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/manage-block-storage.html["Administra el almacenamiento en bloques en NetApp Workload Factory"]



==== Actualización de la función de gestión automática de capacidad

Esta actualización separa la función en dos características distintas: gestión automática de capacidad y notificaciones de capacidad.

La gestión automática de la capacidad añade o elimina automáticamente capacidad del sistema de archivos para mantener un rendimiento óptimo. Las nuevas opciones incluyen los modos incremental y adaptativo. En el modo incremental, el sistema aumenta la capacidad en una cantidad fija en los umbrales; en el modo adaptativo, usa datos históricos para predecir y ajustar la capacidad.

Las notificaciones de capacidad te avisan cada vez que ocurre un ajuste de capacidad y cuando los sistemas de archivos alcanzan los límites establecidos, así puedes gestionar el almacenamiento antes de que surjan problemas.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/enable-auto-capacity-management.html["Conoce la gestión automática de la capacidad y las notificaciones de capacidad en NetApp Workload Factory"]

=== Cargas de trabajo EDA



==== Panel de control general para la gestión del almacenamiento EDA

Como administrador de TI que gestiona docenas de sistemas de archivos FSx para ONTAP, puedes usar el panel de información general para responder a preguntas operativas comunes sobre el estado del clúster, la utilización y la colocación de volúmenes.

El panel de resumen incluye:

* *Estado de salud del clúster*: Información resumida en la parte superior del panel de control que destaca el rendimiento, la capacidad, la disponibilidad, la protección y los eventos de seguridad en todos tus clústeres.
* *Tabla de clústeres*: Una tabla con búsqueda, filtrado y paginación que muestra la capacidad SSD, el grupo de capacidad y las métricas de rendimiento por clúster, con soporte para la personalización de columnas y exportación CSV.


El anterior cuadro de mandos principal ahora es accesible en la pestaña *Project*. Las pestañas EDA ahora están ordenadas: visión general, latencia, Project, CI/CD.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html["Más información sobre el dashboard de resumen"^].



==== Análisis automatizado de latencia

El análisis de latencia ahora incluye un análisis básico automatizado. Cuando se detectan eventos de latencia, el sistema analiza automáticamente las métricas de delay center de ONTAP QoS para identificar el origen de los problemas de latencia.

El análisis básico evalúa las políticas de QoS, el rendimiento de los discos, los problemas de red y las configuraciones de clúster para proporcionar información inmediata en la columna Details de la tabla de eventos de latencia.

=== Amazon FSX para ONTAP de NetApp



==== Soporte para la migración de storage VM

NetApp Workload Factory ahora admite la migración para storage VMs. Esta función permite la migración de datos y configuraciones de sistemas de almacenamiento ONTAP desde sistemas ONTAP locales o sistemas de archivos FSx for ONTAP de primera generación a sistemas de archivos FSx for ONTAP de segunda generación. Puedes replicar datos y configuraciones de storage VM para trasladarlos al nuevo sistema de archivos con un tiempo de inactividad y una interrupción mínimos para los usuarios y las aplicaciones.

Para utilizar esta función, link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["crear una relación de replicación"] y selecciona *Migration* como caso de uso. Para completar el proceso de migración, debes link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html#cut-over-replication-for-migration-use-cases["transiciona la máquina virtual de almacenamiento y sus volúmenes replicados"] inmediatamente para migrar de forma permanente los datos y la configuración de la VM de almacenamiento al sistema de archivos FSx for ONTAP de destino.
