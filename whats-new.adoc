=== Cargas de trabajo de base de datos



==== Ahorro personalizado para hosts Oracle en Amazon Elastic Block Store

NetApp Workload Factory for Databases te permite personalizar el ahorro para hosts Oracle en Amazon Elastic Block Store (EBS). Puedes ajustar varios campos como la frecuencia de las instantáneas, el número de copias clonadas y los detalles del volumen para ver cómo afectan a tu ahorro.

NetApp Workload Factory para bases de datos ahora te permite seleccionar varios grupos de seguridad al desplegar hosts de Microsoft SQL Server y PostgreSQL. Esto te da mayor flexibilidad al configurar la seguridad de red para tus cargas de trabajo de bases de datos.

* link:https://docs.netapp.com/us-en/workload-databases/create-database-server.html["Implementa hosts de Microsoft SQL Server"]
* link:https://docs.netapp.com/us-en/workload-databases/create-postgresql-server.html["Implementa hosts PostgreSQL"]

=== Amazon FSX para ONTAP de NetApp



==== Soporte para AWS GovCloud (US) en NetApp Workload Factory

NetApp Workload Factory es compatible con las cuentas de AWS GovCloud (US). Con una cuenta de GovCloud, puedes implementar y administrar cargas de trabajo de almacenamiento en las regiones de AWS GovCloud (US).



==== Establece umbrales de latencia de volumen en NetApp Workload Factory

Puedes configurar umbrales de latencia por volumen para supervisar el rendimiento y recibir alertas cuando los valores de latencia superen tus límites definidos. Workload Factory supervisa la latencia tanto de lectura como de escritura y genera alertas cuando se supera cualquiera de los dos umbrales.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["Establece umbrales de latencia de volumen en NetApp Workload Factory"]



==== Haz un seguimiento de los retrasos de replicación en NetApp Workload Factory

Con NetApp Workload Factory, puedes hacer un seguimiento de los retrasos de replicación de tus sistemas de archivos FSx para ONTAP que están configurados con replicación remota. Los retrasos de replicación indican el tiempo de retraso entre cuando los datos se escriben en el sistema de archivos de origen y cuando se replican en el sistema de archivos de destino. Hacer un seguimiento de los retrasos de replicación te ayuda a monitorear el estado de tu configuración de replicación y a identificar posibles problemas que podrían afectar la coherencia de los datos y los objetivos de punto de recuperación (RPO).

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-replication-delays.html["Haz un seguimiento de los retrasos de replicación en NetApp Workload Factory"]



==== Identificación de hosts y definición de cargas de trabajo en NetApp Workload Factory

Workload Factory identifica los hosts que consumen almacenamiento y los tipos de cargas de trabajo que se asocian a la actividad del host. Las definiciones de las cargas de trabajo se generan mediante IA a partir de las etiquetas y los metadatos de AWS. Puedes ver los detalles del host y las definiciones de las cargas de trabajo desde las pantallas de recursos de sistema de archivos, volumen y dispositivo de bloques en Workload Factory.



==== Detalles del LIF disponibles en NetApp Workload Factory

Mostramos todos los puntos finales del sistema de archivos y de las máquinas virtuales de almacenamiento directamente en la pestaña Overview de un sistema de archivos. La visibilidad directa de los puntos finales dentro de la consola de Workload Factory facilita supervisar el acceso a la red, los puntos finales y el estado de la conmutación por error sin usar la CLI o CloudShell. Para mostrar los detalles de LIF, incluidas las funciones, los puntos finales, los nodos, las direcciones IP y el estado, necesitas asociar un vínculo con el sistema de archivos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["Aprende cómo asociar un enlace con tu sistema de archivos en NetApp Workload Factory"].



==== Mejoras de costes

Hay disponibles nuevos filtros opcionales a nivel de página para mejorar los resultados del análisis de costes de tus recursos en todas las cuentas de AWS. Puedes seleccionar varias regiones y usar la clave de etiqueta de asignación del sistema de archivos y los valores clave para filtrar los resultados del análisis de costes y obtener detalles de costes específicos de los recursos para tus sistemas de archivos y volúmenes FSx para ONTAP.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-costs.html["Usa filtros para el análisis de costes en NetApp Workload Factory"]



==== Programación de la inicialización de la replicación disponible en NetApp Workload Factory

Durante la configuración de la replicación de datos, ahora puedes programar la inicialización de la replicación para tus sistemas de archivos FSx para ONTAP. Las opciones son inicializar la relación de replicación inmediatamente o a una hora determinada. La inicialización de la replicación es el proceso de creación de una transferencia completa para la replicación mediante la copia de datos desde el sistema de archivos de origen al sistema de archivos de destino. Programar la inicialización de la replicación te permite automatizar este proceso y asegurarte de que tu configuración de replicación está actualizada sin intervención manual.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["Replica datos en NetApp Workload Factory"]



==== Informe en Excel del estado bien arquitectado de FSx para sistemas de archivos ONTAP

Después del análisis, puedes descargar un informe de Excel sobre el estado bien arquitectado de tus sistemas de archivos FSx for ONTAP para tus registros o para compartirlo con tus equipos.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["Consulta el estado bien diseñado de tus sistemas de archivos FSx para ONTAP en NetApp Workload Factory"]



==== Ver y analizar eventos de latencia

Workload Factory ahora admite la visualización y el análisis de eventos de latencia para volúmenes FSx para ONTAP. Puedes ver eventos de latencia sin una conexión de enlace, lo que facilita la supervisión de problemas de rendimiento. Para analizar eventos de latencia y recibir información y recomendaciones impulsadas por IA, debes conectarte al sistema de archivos usando un enlace. El análisis proporciona valores de latencia actuales, umbrales, análisis de respuesta de QoS y orientación para la corrección.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/analyze-events.html["Analiza los eventos de FSx for ONTAP en NetApp Workload Factory"]



==== Establecer umbrales de latencia de volumen

Ahora puedes configurar umbrales de latencia por volumen para supervisar el rendimiento y recibir alertas cuando la latencia de lectura/escritura supere los límites que hayas definido. La supervisión de la latencia te ayuda a identificar de forma proactiva los problemas de rendimiento antes de que afecten a tus cargas de trabajo.

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["Establece umbrales de latencia de volumen en NetApp Workload Factory"]

==== Análisis bien diseñado para Oracle

El análisis bien diseñado incluye las siguientes nuevas configuraciones para Oracle:

* *Configuración de backup*: comprueba si tus bases de datos Oracle están protegidas por AWS Backup o FSx for ONTAP backups.
* *Limpieza de clones*: recomienda eliminar los clones obsoletos o no utilizados para reducir los costos de almacenamiento.

==== Cálculo automático de ahorros para bases de datos Oracle en Amazon Elastic Block Store (EBS)

Workload Factory analiza automáticamente el coste del almacenamiento EBS utilizado para las bases de datos Oracle y luego sugiere la mejor y más rentable configuración de FSx para ONTAP. Si es necesario, la carga de trabajo de base de datos te redirige al asistente de replicación de la carga de trabajo de almacenamiento de FSx for ONTAP para gestionar los requisitos previos del enlace del mismo modo que lo hace Storage.

=== Cargas de trabajo EDA



==== Notificaciones por correo electrónico y SNS para eventos de latencia

Ahora puedes recibir notificaciones automáticas cuando se detecten eventos de latencia en tus volúmenes FSx para ONTAP. Configura las notificaciones por correo electrónico o Amazon SNS en los ajustes de Workload Factory para recibir alertas en tiempo real cuando los volúmenes superen tus umbrales de latencia configurados.

Las notificaciones se envían por sistema de archivos e incluyen detalles sobre todos los volúmenes afectados, niveles de gravedad, comparaciones de umbrales y un enlace directo a la página de latencia para su investigación.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["Aprende a configurar las notificaciones de latencia"^].



==== Visualización de tendencias de latencia con gráficos interactivos

El panel de análisis de latencia ahora incluye una interfaz con pestañas para un análisis exhaustivo de la latencia:

* *Resumen*: Ver los resultados del análisis básico automatizado que muestra qué componente está causando la latencia
* *A lo largo del tiempo*: Visualiza el comportamiento de la latencia del volumen con un gráfico interactivo a lo largo de múltiples marcos temporales (1 hora, 3 horas, 12 horas, 24 horas o 72 horas)


La pestaña *Over time* muestra los datos brutos de latencia CloudWatch (lectura o escritura, dependiendo de qué alarma se activó) con líneas de umbral de advertencia y crítico, indicadores de incumplimiento y métricas detalladas para cada infracción.

Esta completa visualización te ayuda a identificar patrones, entender si los problemas de latencia son recurrentes o aislados y tomar decisiones más informadas sobre la corrección.

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["Aprende a analizar las tendencias de latencia"^].

=== Cargas de trabajo EDA



==== Recomendaciones de capacidad de EDA

Workload Factory ahora analiza automáticamente la capacidad SSD de cada clúster FSx para ONTAP en tu inventario EDA y genera recomendaciones de capacidad cuando se necesitan ajustes. El algoritmo se ejecuta una vez cada 24 horas para cada clúster.

Cuando se identifica una recomendación:

* Recibes una notificación inmediata por correo electrónico o WAD (no forma parte del resumen semanal).
* Aparece un indicador luminoso en el cluster afectado en la tabla Clusters.
* Un resumen en el panel de descripción general muestra el número total de clusters con recomendaciones activas.


Selecciona un cluster con una recomendación activa para ver la recomendación, que explica el cambio sugerido y el razonamiento.

https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#capacity-recommendations["Más información sobre las recomendaciones de capacidad"^].



==== Análisis automatizado de latencia de IA

El análisis de latencia ahora incluye el análisis de agentes de IA.

Para escenarios complejos que impliquen latencia de datos o de clúster, puedes invocar el análisis de agentes de IA para recibir:

* Identificación detallada de la causa raíz
* Lista de clientes EC2 afectados
* Medidas correctoras específicas


El análisis del agente de IA requiere la configuración de ARN del modelo de Amazon Bedrock y está disponible bajo demanda para escenarios en los que el análisis básico recomienda una investigación más profunda. El agente de IA puede identificar problemas como volúmenes bully, configuraciones de puntos de montaje no óptimas, necesidades de reequilibrio de FlexGroup y requisitos de escalado horizontal.

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#run-ai-agent-analysis["Más información sobre el análisis de agentes de IA"^].

=== Cargas de trabajo de



==== Presentamos la compatibilidad del asesor de migración para Google Cloud VMware Engine

NetApp Workload Factory ahora es compatible con Google Cloud VMware Engine. Puedes migrar tus cargas de trabajo VMware locales a Google Cloud VMware Engine usando Google Cloud NetApp Volumes como almacenes de datos NFS externos, optimizando los costes y permitiendo más control sobre tu entorno VMware sin necesidad de refactorizar o cambiar de plataforma tus aplicaciones.

El asesor de migración de Google Cloud VMware Engine proporciona:

* Dimensionamiento y diseño automatizados del clúster Google Cloud VMware Engine con compatibilidad con los tipos de nodo VE1 y VE2
* Optimización de la disposición de las máquinas virtuales entre los datastores internos de vSAN y los datastores NFS externos de Google Cloud NetApp Volumes.
* Comparación detallada de costes y análisis del coste total de propiedad entre las configuraciones solo vSAN y Google Cloud NetApp Volumes-enhanced.
* Soporte para los niveles de servicio Standard, Premium y Extreme de Google Cloud NetApp Volumes.
* Despliegue de Infrastructure-as-Code con Terraform para el aprovisionamiento automatizado.
